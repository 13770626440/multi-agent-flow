# Multi-Agent Flow 项目

## 项目概述

**Multi-Agent Flow** 是一个基于 OpenClaw 2026.4.5 的 Subagent 框架，实现企业级的多智能体协作系统。

### 核心目标
1. **助理 Agent + Subagent 团队管理**：一个助理 Agent 负责对话，创建/管理/维护/销毁由多个 subagent 组成的团队
2. **任务计划模板系统**：任务目标定义、任务分解、执行监控与风险异常反馈
3. **知识库 + 自反馈优化**：运行结果自动沉淀为 skill 和优化模板
4. **Dashboard**：监控运行情况、subagent 生命周期、skill 情况

### 技术架构
- **自研·规划层**：模板管理、任务分解、复盘优化、Skill沉淀、Agency-Agent管理
- **OpenMOSS**：任务状态机、Agent调度、审查返工、巡查、Dashboard
- **OpenClaw Lobster**：Agent生命周期、并行执行、工具调用

### 核心差异化
- **模板系统** + **自反馈优化闭环** + **Agency-Agent职业化体系**（所有现有框架都没有）

- 所有代码必须归档到 `D:\coding\multi-agent-flow\code`

- 所有文档撰写必须先经过项目负责人确认
- 所有代码开发必须先经过项目负责人确认

### 3. 评审机制
建立三角色评审组：
- **架构师**：技术/架构角度，关注实现可行性和成本（使用 GLM-5 模型）
- **产品经理**：产品角度（使用 kimi2.5 模型）
- **项目经理**：项目管理/执行角度（使用 DeepSeek3.1 模型）

**评审标准**：
- 评审结论打分必须 > 9分/10分
- 所有 P0/P1 问题必须修复
- 评审通过后方可进入下一阶段
